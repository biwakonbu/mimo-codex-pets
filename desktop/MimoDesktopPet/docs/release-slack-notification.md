# Release Slack Notification

Mimo Desktop Pet posts a deploy-success notification to Slack from GitHub
Actions when a GitHub Release is published.

This notification is intentionally sent by GitHub Actions, not by a local Codex
session, so the message reflects the repository's deployment event.

## Trigger

The workflow is:

```text
.github/workflows/release-slack-notify.yml
```

It runs on:

- `release.published`: the normal deploy-success path
- `workflow_dispatch`: manual re-send for an existing release tag

For this repository, "deployed" means the release wrapper has created and
published a GitHub Release with the notarized DMG asset attached.

## Required Secret

Create a Slack Incoming Webhook for the `#mimo-desktop` channel, then add its
URL as this GitHub Actions repository secret:

```text
MIMO_DESKTOP_SLACK_WEBHOOK_URL
```

Do not commit the webhook URL. Treat it like a credential; rotate it if it is
ever pasted into chat, logs, or source files.

## Message Shape

The GitHub Action builds a Slack Block Kit payload with:

- deploy-success header
- version tag
- release link
- DMG download button
- repository button
- DMG file name and size
- SHA-256 digest when GitHub exposes the asset digest
- published timestamp and target branch/commitish

The payload is generated by:

```text
desktop/MimoDesktopPet/script/build_release_slack_payload.py
```

The action fails instead of sending a misleading success message when:

- the release is still a draft
- the release is marked as a prerelease
- the release has no `.dmg` asset
- the Slack webhook secret is missing

## Manual Re-Send

After the secret is configured, re-send the notification for an existing release
from the repository root:

```bash
gh workflow run release-slack-notify.yml -f tag=v0.0.1
```

Then watch the run:

```bash
gh run list --workflow release-slack-notify.yml --limit 1
```

## Local Payload Check

You can inspect the message payload without sending anything:

```bash
gh release view v0.0.1 \
  --json tagName,name,url,isDraft,isPrerelease,publishedAt,targetCommitish,assets \
  > /tmp/mimo-release.json

python3 desktop/MimoDesktopPet/script/build_release_slack_payload.py \
  --release-json /tmp/mimo-release.json \
  --repository biwakonbu/mimo-codex-pets \
  --repository-url https://github.com/biwakonbu/mimo-codex-pets \
  --output /tmp/mimo-slack-payload.json

python3 -m json.tool /tmp/mimo-slack-payload.json
```
